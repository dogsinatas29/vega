# 🗺️ VEGA 개발 로드맵 (ROADMAP)

이 문서는 VEGA의 남은 작업과 향후 개발 계획을 관리합니다.

## 🚀 우선순위: SSH 연결 고도화
사용자 의견 및 현재 구현 상태를 바탕으로 다음 작업을 우선 진행합니다:

- [ ] **SSH 키 관리**: `vega setup` 시 커스텀 SSH 키 경로 지정 및 관리 기능 추가.
- [ ] **보안 정책 제어**: 환경에 따른 `StrictHostKeyChecking` 옵션 조절 기능.
- [ ] **가변 타임아웃**: 네트워크 지연 시간에 따른 지능적 SSH 타임아웃 조정.
- [ ] **연결 멀티플렉싱**: 반복 명령 실행 속도 향상을 위한 `ControlMaster` 지원.

## 🏢 Phase 5: 엔터프라이즈 (Cloud & Persistence)
*목표: 다중 환경 지원 및 장기 리포팅 시스템 구축*

- [ ] **rclone 클라우드 동기화**:
    - `rclone`을 활용한 구글 드라이브/S3로의 프로젝트 자동 백업.
    - 기기간 `vega.db` 상태 동기화.
- [ ] **영구 메타데이터 저장소**:
    - 장기 설정을 위한 SQLite 내 전용 `metadata` 테이블 구축.
    - 로컬 키링 기반의 데이터 암호화 저장.
- [ ] **고급 리포팅 엔진**:
    - SRE 활동에 대한 PDF/Markdown 보고서 내보내기.
    - AI 요약을 통한 자동 이메일/슬랙 알림 발송.

## 🛡️ Phase 6: 자율 복구 강화 (Planned)
*목표: 극한 환경 및 오프라인 대응력 강화*

- [ ] **오프라인 진단 룰셋**: AI 연결 불가 시 작동하는 정규표현식/규칙 기반 엔진 강화.
- [ ] **GRUB 복구 모드 가이드**: 부팅 문제 발생 시 맥락에 맞는 수동 복구 안내.
- [ ] **자동 의존성 치유**: 누락된 시스템 도구의 선제적 탐지 및 자동 설치.
